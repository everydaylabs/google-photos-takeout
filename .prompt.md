# Google Photos Takeout Project Enhancement Prompt

## Project Overview
You are working with a Python-based Google Photos Takeout processing system that handles large-scale photo/video exports (200+ GB). The project consists of three main scripts that work in sequence to analyze, organize, and cleanup Google Photos takeout data.

## Current Architecture
The project has three core Python scripts:
1. **`analyze_takeout.py`** - Scans and categorizes media files, generates metrics
2. **`organize_takeout.py`** - Organizes files by date using EXIF metadata into year/month/day structure
3. **`cleanup_takeout.py`** - Removes empty folders and processes remaining files

## Key Issues Identified

### 1. Critical Bug: File Overwriting Risk
- In `organize_takeout.py` line 58, there's a bug where files are renamed with UUIDs but the original filename is lost
- The line `new_filenpath = f"{uuid.uuid4()}{extension}"` creates a new variable but then `os.path.basename(new_filenpath)` is used, which would cause issues
- This could lead to data loss or unexpected behavior

### 2. Code Quality Issues
- Inconsistent error handling across scripts
- Some functions lack proper input validation
- Logging could be more structured and informative
- Missing type hints throughout the codebase
- Some hardcoded values that should be configurable

### 3. Performance Concerns
- EXIF processing is done sequentially in some cases where it could be parallelized
- File operations could benefit from better batching
- Memory usage could be optimized for large file sets
